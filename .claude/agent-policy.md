# AI Agent 行为协议（Agent Operating Policy）

## 目标

在尽可能减少人工干预的前提下，持续自主推进当前任务。

Agent 应具备：

- 自主分析
- 自主执行
- 自主验证
- 自主修复
- 自主汇报

但必须严格遵守风险分级与人工升级机制。

***

# 决策分级体系

## 🟢 绿灯决策（允许完全自主执行）

以下操作允许 AI 直接执行，无需人工确认。

### 示例

- 查阅本仓库文档（README、CLAUDE、doc/notes）
- 搜索和阅读 Clojure / Python 源码
- 修改 `src/` 下工具逻辑与重构命名空间
- 补充或调整 `test/` 下测试并执行 `lein test`
- 修复 `lein` 构建失败、语法错误、依赖冲突（不新增依赖）
- 调整 `resources/templates/*.ini` 的模板内容（小范围、可回滚）
- 更新 `doc/`、`CHANGELOG.md`、`.claude/loop.md` 等文档
- 增加日志输出、注释与可观测性信息
- 运行只读分析（如 ID 占用扫描、文本检索、静态检查）
- 失败后的自动重试（同一方案最多 3 次）

### 执行规则

- 允许直接执行
- 无需中断任务
- 在下一次汇报中说明即可

***

## 🟡 黄灯决策（允许执行，但必须汇报）

以下操作允许 AI 自主执行，但执行后必须明确汇报：

### 示例

- 新增或升级依赖（`project.clj` / JitPack 相关）
- 修改 `resources/config.edn` 的目录配置（`:project-dir`、`:workspace`、`:out-dir`、`:temp-dir`）
- 变更 `gsein-war3.http.server` 行为（端口、路由、WebSocket、启动方式）
- 新增对外部工具或服务的调用（网络请求、外部 API、远程存储）
- 调整批量生成策略（单位/物品/技能/任务生成器规则）
- 批量改写大量地图资源文件（LNI / MDX / BLP），影响范围跨多个目录
- 修改 Python 工具对外输出格式（如彩色文本编码规则）
- 变更打包与发布产物策略（`lein jar` / `uberjar` 产物用途）
- 需要人工配合的环境变更（新增环境变量、系统级依赖）
- 创建或更新 PR、发布说明、协作分支策略

### 执行规则

执行后必须汇报：

1. 做了什么
2. 为什么这样做
3. 影响范围
4. 潜在风险
5. 回滚方案

***

## 🔴 红灯决策（必须停止并等待人工确认）

以下操作禁止 AI 自动执行。

必须暂停任务并等待人工确认。

### 示例

- 删除或覆盖真实地图工程目录中的关键资源（不可恢复）
- 对外发布或替换线上使用的地图包、脚本包、配置包
- 执行不可逆批处理（大规模删除、覆盖、重命名且无备份）
- 修改或泄露密钥、令牌、证书、私有仓库凭据
- 将本地 HTTP 服务暴露到公网，或开放未审计接口
- 执行高风险 Shell 命令（批量删除、系统级权限变更、网络策略变更）
- 在未确认范围时直接改写用户指定的真实 War3 项目目录
- 任何涉及法律、版权、合规风险的对外分发或声明
- 缺少业务确认却进行架构级重写或长期方向调整
- 其他可能影响真实用户资产且无法快速回滚的操作

### 执行规则

必须立即停止。

并输出：

1. 准备执行的操作
2. 为什么属于红灯
3. 风险说明
4. 可选方案
5. 预期影响

等待人工明确批准后才能继续。

***

# 风险升级规则

如果无法明确判断属于哪个等级：

默认升级到更高风险等级。

禁止 AI 自动降低风险等级。

***

# 失败处理机制

如果任务失败：

1. 自动分析原因
2. 自动读取日志
3. 自动查阅文档
4. 自动尝试修复
5. 自动重试

连续失败超过 3 次：

- 停止继续重试
- 总结失败原因
- 提出建议方案
- 请求人工介入

***

# 长任务执行规则

AI 应将大型任务拆分为多个“小闭环”。

每轮循环只完成：

- 一个明确目标
- 一个可验证结果
- 一个最小变更集

每轮都必须：

1. 分析当前状态
2. 选择下一个任务
3. 执行修改
4. 验证结果
5. 更新任务状态
6. 生成进度汇报

***

# 汇报格式

每轮循环结束时输出：

## 当前进展

- 已完成：
- 当前进行中：
- 下一步计划：

## 风险情况

- 新风险：
- 是否涉及黄灯：
- 是否触发红灯：

## 修改内容

- 修改文件：
- 新增文件：
- 运行结果：

***

# 人工介入条件

出现以下情况时必须请求人工介入：

- 红灯决策
- 连续失败超过 3 次
- 缺少必要业务信息
- 需求存在歧义
- 涉及真实资金
- 涉及真实用户
- 涉及生产环境
- 涉及安全 / 法律 / 合规

***

# 核心原则

AI 的首要目标：

不是“尽可能自动化”。

而是：

“在安全边界内持续推进任务”。

宁可暂停等待确认：

也不要越权执行危险操作。
